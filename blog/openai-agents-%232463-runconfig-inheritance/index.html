Fix RunConfig Inheritance for Agent-as-Tool in OpenAI Agents SDK

This post covers a core runtime fix in OpenAI's Agents SDK, the official Python framework for building agent applications.

Primary References

  • Pull Request: openai/openai-agents-python#2463
  • Reviewer: @seratch
  • Files changed: 6 files, +259/-2

Architecture Overview

The Problem

In the "agent-as-tool" pattern, a parent agent invokes a child agent as a tool. The child agent should inherit the parent's RunConfig (tracing settings, model_provider, privacy configs). But ToolContext had no run_config field, so the child always used default settings.

Key Changes

  1. ToolContext: added run_config: RunConfig | None field
  2. tool_execution.py: passes active run_config into ToolContext
  3. Agent.as_tool(): inherits parent's RunConfig when none is explicitly provided
  4. Made run_config keyword-only per maintainer CR feedback
# Agent.as_tool() — inherit parent RunConfig
async def run_agent_tool(ctx: ToolContext, input: str):
    config = explicit_config or ctx.run_config  # ← new inheritance
    return await Runner.run(child_agent, input, run_config=config)

Test Coverage

Added tests across 3 files covering inheritance, override, and propagation scenarios — 259 lines added.

Impact

  • Fixes configuration consistency in multi-agent chains — the core use case of the Agents SDK
  • Ensures tracing, privacy, and model_provider settings propagate correctly
  • 259 additions across 6 files — the largest single contribution to the SDK in this batch

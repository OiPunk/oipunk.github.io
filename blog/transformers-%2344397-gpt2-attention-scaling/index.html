Fix GPT-2 Attention Scaling Ignored in SDPA/FlashAttention

This post covers my merged fix in Hugging Face Transformers (140K+ stars), addressing a silent correctness bug in GPT-2's attention layer.

Primary References

  • Pull Request: huggingface/transformers#44397
  • Reviewer: @vasqu
  • Files changed: 3 files, +86/-50

Bug Flow Overview

The Problem

GPT2Attention.forward() never passed the scaling parameter to attention_interface(). This meant two GPT-2 configuration options were silently ignored when using SDPA or FlashAttention:

  • scale_attn_weights — controls whether attention weights are scaled by 1/√d
  • scale_attn_by_inverse_layer_idx — scales attention by 1/(layer_idx+1)

Only the eager attention path applied these correctly, because it checked module.scale_attn_weights internally. SDPA and FlashAttention relied on the scaling parameter — which was never passed.

Why This Was Hard to Catch

This is a silent correctness bug: the code runs without errors, but produces subtly wrong outputs. The default scaling (1/√d) is often close enough to not cause obvious failures, making this nearly invisible in standard testing.

The Fix

I refactored the attention scaling to be consistent across all backends:

# In __init__: precompute unified scaling factor
self.scaling = self.head_dim ** -0.5
if self.scale_attn_by_inverse_layer_idx:
    self.scaling /= float(layer_idx + 1)
if not self.scale_attn_weights:
    self.scaling = 1.0

# In forward: pass to all backends
attn_output = attention_interface(
    ...,
    scaling=self.scaling,  # ← now all backends use correct scaling
)

I also refactored eager_attention_forward to accept scaling and dropout as parameters (Bert-style), removing the internal module attribute checks.
